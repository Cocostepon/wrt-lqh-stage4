# 合并测试用例（merged_test_cases.md）

说明：本表整合了项目中所有测试场景，覆盖正常流程、多轮记忆、参数缺失、模糊指令、复杂任务（Planner→Orchestrator）、LLM 输出异常等。建议在每条用例执行后在“状态”一栏记录结果（例如：✅ 通过 / ❌ 未通过 / 待测）。

| 编号  | 测试类型              | 用户输入 (Input)                | 预期输出 (Expected Output)                                                   | 备注 / 状态 |
| ---:|:----------------- |:--------------------------- |:------------------------------------------------------------------------ |:------- |
| 01  | 查单词（正常）           | 查单词 agent                   | 返回 Agent 的词义，格式例如：`【查词助手】agent: 代理人；智能体`                                 | ✅ 通过    |
| 02  | 查单词（缺参）           | 查单词                         | 提示“请输入要查询的单词。”（主动追问缺失参数）                                                 | ✅ 通过    |
| 03  | 算题（正常）            | 算 12*34                     | 返回正确结果，例如：`【算题助手】计算结果为：408`                                              | ✅ 通过    |
| 04  | 算题（非法）            | 算 苹果+桔子                     | 不崩溃，提示输入格式错误，例如：`【错误】数学算式格式不正确` 或要求输入正确算式                                |         |
| 05  | 记笔记（正常）           | 保存笔记: 今天学习了AI Agent         | 返回 `【笔记助手】记录成功！` 并将内容追加写入 `notes.txt`                                    |         |
| 06  | 多轮记忆（问复数）         | 1) 查 agent  2) 那它的复数形式呢？    | 能结合第一轮对话理解上下文，第二步实际查询 “Agents” 并返回结果                                     |         |
| 07  | 多轮任务切换            | 1) 算 5+5  2) 把这个结果记下来       | Agent 记住上一轮结果（10），并将其保存到笔记，返回保存成功提示                                      |         |
| 08  | 模糊指令（功能未明）        | 帮我查点东西                      | Agent 询问澄清：要查单词、算题还是其他？（主动追问并等待确认）                                       |         |
| 09  | 复杂需求（规划）          | 生成本周学习周报                    | Planner 输出步骤列表（JSON），Orchestrator 依次执行并整合结果，最终返回周报概要（并可保存）               |         |
| 10  | LLM 输出异常 / 大文本鲁棒性 | 提示一个超长复杂输入或特殊字符             | Planner/Orchestrator/主程序能捕获异常（try/except），返回清晰错误提示而非崩溃；并把原始 LLM 输出记录以便调试 |         |
| 11  | 主动追问（缺少参数）        | 我想记个笔记                      | Agent 识别缺少笔记内容，主动询问：“请告诉我您想记录的具体内容。”                                     |         |
| 12  | 参数上下文引用           | 1) 算 6*7  2) 把上次结果乘 3       | 使用上下文：上次结果 42 -> 计算 42*3 并保存/返回结果                                        |         |
| 13  | 系统鲁棒性（边界/异常）      | 输入超出长度限制的文本或特殊符号            | 程序通过异常处理返回友好提示（例如：“连接大脑失败”或“输入格式不支持”），不崩溃                                |         |
| 14  | 多步组合示例（生成+保存）     | 生成周报并保存为 notes.txt          | Planner 先生成文本草稿（或由 Orchestrator 调用 LLM 生成），然后调用 save_note 写入并返回成功        |         |
| 15  | 并发/顺序依赖（可选高级）     | 多个相互依赖的步骤（如：查词 -> 翻译 -> 保存） | Orchestrator 能按顺序传递 context 并整合最终输出（自然语言）                                |         |
